Stacks

The gds_env provides a Jupyter-based platform to run Python and R geospatial packages in a way that can be installed in a variety of contexts. To provide more flexibility and better cater to the needs of different users, this overall functionality is stratified in three different flavours, or stacks: gds_py, gds and gds_dev.

All of the stacks are based on Jupyter Lab/Notebook/Hub and on Docker. The main difference is the overall functionality they provide (and the subsequent size of each image).

The three stacks are layered on top of each other:

  • gds_py: Jupyter minimal-notebook (Jupyter Lab/Notebook/Hub + LaTeX) + Python geospatial libraries (see full list)
  • gds: gds_py + R geospatial libraries (see full list) + IRkernel + rpy2
  • gds_dev: gds + development tools (decktape, jekyll, tippecanoe)

For further details, be sure to check the page of each stack.
